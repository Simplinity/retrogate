# Add Update Checker via GitHub Releases API

## Context
RetroGate v1.1.0 is distributed via GitHub releases (`github.com/Simplinity/retrogate`), with no update mechanism. Users have no way to know when a new version is available. This adds a lightweight check-on-launch feature using the GitHub Releases API ‚Äî no extra dependencies, no auto-install.

## How it works
1. On launch (2s delay), call `https://api.github.com/repos/Simplinity/retrogate/releases/latest`
2. Parse `tag_name` (e.g. `"v1.2.0"`), compare with `CFBundleShortVersionString` from Bundle.main
3. If newer version exists ‚Üí show NSAlert with "Download" / "Later" buttons
4. "Download" opens the GitHub release page in Safari
5. Fail silently on any error (network, parse, rate limit)

## Design decisions
- **No persistence**: Check every launch, no "skip version" or "last check date". Simple, releases are infrequent.
- **NSAlert, not SwiftUI `.alert`**: One-shot modal, no state binding needed, can show multi-line release notes.
- **Separate file**: RetroGateApp.swift is already 1800+ lines. Self-contained concern.
- **`enum` with static method**: No state to maintain, no accidental instantiation.
- **`hasChecked` guard**: Prevents re-check if user closes and reopens window.
- **2s delay**: Avoids competing with onboarding sheet or window animation.
- **SPM builds**: `Bundle.main.infoDictionary` is nil in SPM debug builds ‚Üí check silently skips. Desirable.
